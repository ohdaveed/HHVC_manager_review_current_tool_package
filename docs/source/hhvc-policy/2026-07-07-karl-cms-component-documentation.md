# Karl CMS Component Documentation (Table 1 extract)

Source: `2026-07-07-karl-cms-component-documentation.xlsx`

For per-content-type Karl editor field order, SF.gov screenshot mapping, and HHVC mockup schema alignment, see [karl-content-type-field-reference.md](./karl-content-type-field-reference.md). This component extract describes _what each component does_; the field reference describes _which page types expose which fields_.

## 1. Accordions

**Description:** Component that allows users to show and hide sections of related content on a page.

**Usage guidelines:** Use for FAQs or supplemental info. Limit to groups of 5. Never hide critical instructions, fees, deadlines, or report paths.

**GitBook slug:** sfdigitalservices.gitbook.io/.../accordions

**Source index:** 1-25

## 2. Address

**Description:** A reusable component for adding physical locations that live in a central library.

**Usage guidelines:** Relational content; changes made via Snippets menu update everywhere the address displays.

**GitBook slug:** sfdigitalservices.gitbook.io/.../address

**Source index:** 1-5, 7-9, 14-17, 26-28

## 3. Alert

**Description:** A banner across the top of a page for routine (blue) or critical (red) messages.

**Usage guidelines:** Limited to Agency and Location types. Requires start/expiration dates. Use for urgent service notices; avoid for routine guidance.

**GitBook slug:** sfdigitalservices.gitbook.io/.../alert

**Source index:** 1-3, 5-24, 29

## 4. Body, Main body, Text and title

**Description:** Primary component for adding and formatting the main text content of a page.

**Usage guidelines:** Use Heading 3 or 4. Keep sentences under 15 words and paragraphs under 3 sentences. Use Document picker for files.

**GitBook slug:** sfdigitalservices.gitbook.io/.../body-main-body-text-and-title

**Source index:** 1-3, 5, 7, 8, 13-18, 25, 27, 30

## 5. Button

**Description:** A clickable area used to direct users to specific actions or transactions.

**Usage guidelines:** Max 25 characters. Use active, verb-first language (e.g., "Apply"). Recommend no more than one per page.

**GitBook slug:** sfdigitalservices.gitbook.io/.../button

**Source index:** 1-5, 7-9, 13-20, 22, 24, 31-34

## 6. Callout

**Description:** An in-body colored box with an "i" icon for emphasizing specific information.

**Usage guidelines:** Used for mandatory notices, fee changes, or routing users before they start a task. Use for stable, not temporary, info.

**GitBook slug:** sfdigitalservices.gitbook.io/.../callout

**Source index:** 1-6, 9, 13-20, 22, 24, 26, 29, 31, 34-38

## 7. Contact section

**Description:** Standardized section for departmental address, phone, email, and social media links.

**Usage guidelines:** Place at the bottom of all transaction and information pages. Do not use custom text blocks for contact info.

**GitBook slug:** sfdigitalservices.gitbook.io/.../contact-section

**Source index:** 1-3, 5, 10-12, 14-20, 24, 39, 40

## 8. Cost

**Description:** An optional field to specify fees, price types, or eligibility restrictions.

**Usage guidelines:** Select from predefined cost types and add descriptions for specific requirements.

**GitBook slug:** sfdigitalservices.gitbook.io/.../cost

**Source index:** 1-3, 5, 14-17, 27

## 9. Data

**Description:** Component for linking to or embedding public dashboards, datasets, and Power BI resources.

**Usage guidelines:** Staff can book office hours with DataSF for help creating dashboard stories.

**GitBook slug:** sfdigitalservices.gitbook.io/.../data

**Source index:** 1-3, 5, 7, 14-17, 41

## 10. Description

**Description:** A brief summary at the top of the page explaining its purpose and target audience.

**Usage guidelines:** Keep under 110 characters. Put important keywords first to assist search engine relevance and skimming.

**GitBook slug:** sfdigitalservices.gitbook.io/.../description

**Source index:** 1-5, 14-17, 30, 32, 37, 42-50

## 11. Heading

**Description:** Structural text anchors used to organize content and help users scan the page.

**Usage guidelines:** Use H2 for main titles; H3 and H4 for subheadings. Must nest sequentially. Avoid questions, puns, or "introductions".

**GitBook slug:** sfdigitalservices.gitbook.io/.../heading

**Source index:** 1-4, 14, 15, 17, 27, 36, 49, 51-54

## 12. Highlights

**Description:** Image-based components used to draw attention to important content near the top of Agency pages.

**Usage guidelines:** Requires exactly 2 or 3 highlights. Images must be min 350 x 200 px (16:9 aspect ratio recommended).

**GitBook slug:** sfdigitalservices.gitbook.io/.../highlights

**Source index:** 1-3, 5, 12, 14-17, 55

## 13. Images

**Description:** Visual photography assets used to enhance messages across supported content types.

**Usage guidelines:** Avoid text on images. Must add descriptive alt text. Keep subjects centered to prevent cropping issues.

**GitBook slug:** sfdigitalservices.gitbook.io/.../images

**Source index:** 1, 3-5, 9, 14, 15, 27, 28, 54, 56-61

## 14. Links

**Description:** Text elements that direct users to internal or external pages or actions.

**Usage guidelines:** Use descriptive text; do not use "click here". Link to homepages over PDFs. External links should explain the destination.

**GitBook slug:** sfdigitalservices.gitbook.io/.../links

**Source index:** 1-5, 9, 14-16, 21, 27, 37, 52, 54, 62-65

## 15. Partner agency

**Description:** An optional component to list additional departments responsible for the page content.

**Usage guidelines:** Displays at the bottom of the page. Multiple agencies can be tagged to link users to relevant collaborators.

**GitBook slug:** sfdigitalservices.gitbook.io/.../partner-agency

**Source index:** 1-3, 5, 14-17, 27, 52, 66, 67

## 16. People section

**Description:** Displays names and photos collected from Profile pages onto Agency or Location pages.

**Usage guidelines:** Profile pages must be created first before they can be added to this section.

**GitBook slug:** sfdigitalservices.gitbook.io/.../people-section

**Source index:** 1-5, 14-17, 68

## 17. Primary agency

**Description:** Required field identifying the main department or entity that supports the page content.

**Usage guidelines:** Tagging identifies the content owner and can auto-apply branding like logos or breadcrumb links.

**GitBook slug:** sfdigitalservices.gitbook.io/.../primary-agency

**Source index:** 1-3, 5, 14-17, 26, 27, 50, 52, 67, 69-74

## 18. Redirect this page to

**Description:** A utility used to send users from one URL to another.

**Usage guidelines:** This component has been disabled in the CMS. Contact Digital Services for manual redirect assistance.

**GitBook slug:** sfdigitalservices.gitbook.io/.../redirect-this-page-to

**Source index:** 1-3, 5, 13-17

## 19. Related

**Description:** Component that allows users to view content related to the current task by tagging pages.

**Usage guidelines:** Use to help users move to the next logical task. Some links populate automatically via Topic tagging; others are manual.

**GitBook slug:** sfdigitalservices.gitbook.io/.../related

**Source index:** 1-3, 5, 8, 14-17, 27, 67, 72, 73, 75, 76

## 20. Resources

**Description:** Links to internal pages, external websites, or downloadable files related to the page content.

**Usage guidelines:** Supported on About, Campaign, Resource collection, and Topic types. Often displayed under subheaders in bodies.

**GitBook slug:** sfdigitalservices.gitbook.io/.../resources

**Source index:** 1-5, 14-17, 41, 50, 77

## 21. Spotlight

**Description:** High-visibility feature section pairing images with text and a link on a colored background.

**Usage guidelines:** Supported on Agency, Campaign, Report, and Topic pages. Images can be full width or half-width side-by-side.

**GitBook slug:** sfdigitalservices.gitbook.io/.../spotlight

**Source index:** 1-5, 12, 14-17

## 22. Tables

**Description:** Component for organizing complex relational data, fee schedules, or timelines.

**Usage guidelines:** Must include header rows. No merged cells. Restrict to 3 columns to ensure mobile accessibility and avoid scrolling.

**GitBook slug:** sfdigitalservices.gitbook.io/.../tables

**Source index:** 1-3, 5, 7-11, 14-16, 18-22, 24, 56, 59, 78, 79

## 23. Title

**Description:** The primary page identifier that determines the URL and search relevance.

**Usage guidelines:** Must be < 65 characters and use sentence case. Start transactions with a verb. It should be the only H1 on the page.

**GitBook slug:** sfdigitalservices.gitbook.io/.../title

**Source index:** 1-5, 14, 15, 17, 21, 27, 28, 30, 32, 42-44, 48, 49, 52, 54, 62, 80-86

## 24. Topics

**Description:** Tagging component that groups pages under thematic hubs to manage information architecture.

**Usage guidelines:** Tagging a page adds it to the Topic hub automatically. Digital Services manages the creation of new Topic pages.

**GitBook slug:** sfdigitalservices.gitbook.io/.../topics

**Source index:** 1-5, 14-17, 27, 50, 72, 73, 87-92

## 25. URLs

**Description:** Structured web addresses that show users where they are on SF.gov.

**Usage guidelines:** Must be lowercase, hyphenated, and avoid numbers. Short URLs for print require Digital Services approval.

**GitBook slug:** sfdigitalservices.gitbook.io/.../urls

**Source index:** 1-5, 14-16, 27, 28, 86, 88, 93

## 26. Videos

**Description:** Component for embedding videos via YouTube or other external hosting platforms.

**Usage guidelines:** SF.gov does not host video files. Must include reviewed and edited transcripts for accessibility.

**GitBook slug:** sfdigitalservices.gitbook.io/.../videos

**Source index:** 1-3, 5, 14-17, 26, 35, 94

## 27. Documents, including PDFs

**Description:** Component for uploading, sharing, and attaching files to pages.

**Usage guidelines:** Include title and description. Label links with file format and size. Provide HTML equivalents for PDF accessibility.

**GitBook slug:** sfdigitalservices.gitbook.io/.../documents-including-pdfs

**Source index:** 95

## 28. Step List

**Description:** In-page component for breaking down short, sequential processes into chronological steps.

**Usage guidelines:** One primary action per step. Use plain language. For multi-stage journeys, use the Step-by-step content type instead.

**GitBook slug:** Not in source

**Source index:** 10, 20, 24, 34

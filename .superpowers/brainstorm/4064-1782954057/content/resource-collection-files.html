How this fits the existing data shape

  • Page cards keep using target: 'ratsPrevent' (opens the mockup page) — unchanged
  • Download cards need a new card shape: no target, instead something like fileType: 'pdf' | 'infographic' + a label/filename. They render with a file icon and "Download/View" affordance instead of a title+arrow.
  • In the sitemap graph, downloads become dead-end leaf nodes off the prevent hub — no outgoing edges, since a PDF doesn't link anywhere. Visually they'd want a square/file glyph instead of the circular page node, so the graph doesn't imply they're pages with their own sub-links.

Worth confirming

  • How many downloads, roughly? 3 shown here as an example — if it's more like 10+, the "Downloads" section may want its own filter/search rather than a flat grid
  • Do downloads get Karl placement tags too (e.g. "Resource Collection item: file")? Keeping that consistent would matter for the review workflow this tool is built around

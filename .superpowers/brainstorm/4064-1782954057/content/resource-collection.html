Prevent hub as a Resource Collection page

"Resource Collection item" is already a recognized Karl placement tag in this tool (see js/page-render.js:35) — it's the pattern SF.gov uses for a page that's mainly a curated list of links, not narrative body copy. That's a better fit for the prevent hub than a generic Information page.

Why this fits

  • The 6 prevent pages are genuinely parallel and self-contained — no sequence, no "pick one path" narrative like reporting has. That's exactly what Resource Collection is for.
  • Reuses a content type this tool's own Karl-tagging already recognizes, instead of inventing a new pattern — less for reviewers to learn, consistent with how other pages already tag their card links.
  • In the sitemap graph this also gives the node a real, distinct visual identity (not just a color I made up) — it can carry the same "Resource collection" badge the tool already uses elsewhere.

To confirm
